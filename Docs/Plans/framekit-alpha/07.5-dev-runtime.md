# 07.5. Runtime de desarrollo

Esta fase prepara el runtime de Studio antes de implementar la CLI de la fase
08. El scanner, el codegen y el watcher pasan a ser módulos TypeScript del
paquete. Studio los consume desde un único proceso de desarrollo.

## Decisiones

- [x] Mantener el descubrimiento, el manifiesto de navegación, el estado del watcher, el debounce y el servidor en memoria.
- [x] Conservar un módulo generado porque Next/Turbopack necesita imports literales de las plantillas.
- [x] Unificar metadatos y loaders en `.framekit/generated/templates.ts`.
- [x] Usar el custom server únicamente durante desarrollo.
- [x] Mantener `output: 'standalone'` para producción y usar el servidor generado por Next.
- [x] Mover la salida de Next a `.framekit/next` mediante `distDir`.

## Estructura

```text
apps/studio/
├── .framekit/
│   ├── generated/templates.ts
│   └── next/
├── src/
├── server.ts
└── next.config.ts

packages/framekit/src/
├── discovery/
├── codegen/
└── dev/
```

## Discovery y codegen

- [x] Reemplazar `generate-template-registry.mjs` y su declaración manual por `find-templates.ts`.
- [x] Mantener las reglas de segmentos, directorios ignorados, detención en `template.tsx` y orden por slug.
- [x] Devolver `slug`, `title`, `segments` y `absolutePath` en memoria.
- [x] Crear `create-template-module.ts` para producir el módulo TypeScript.
- [x] Crear `write-template-module.ts` para escribir únicamente cuando cambia.
- [x] Generar loaders con rutas relativas literales desde `.framekit/generated`.
- [x] Derivar `templateManifest` y `templateRegistry` del arreglo generado `templates`.
- [x] Eliminar `src/.framekit/manifest.ts` y `src/.framekit/registry.ts`.

## Watcher y servidor

- [x] Crear `watch-templates.ts` con `chokidar` y `close()`.
- [x] Observar altas y bajas de `template.tsx` y directorios.
- [x] Ignorar cambios de contenido dentro de un `template.tsx`.
- [x] Conservar debounce de 150 ms, exclusión de ejecuciones simultáneas y regeneración pendiente.
- [x] Crear `create-dev-server.ts` con generación inicial, Next dev, HTTP server y watcher.
- [x] Crear `apps/studio/server.ts` con manejo de `SIGINT` y `SIGTERM`.
- [x] Mantener `--generate` como modo temporal para los scripts de build, test y typecheck hasta la fase 08.
