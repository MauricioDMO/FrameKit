# Integrate Into an Existing Next.js Project

## Alpha release

FrameKit is currently in alpha/prerelease status. The packages are not yet published to npm. This documentation will be updated once publication is confirmed.

## Prerequisites

- Next.js 16 or later (but not Next.js 17).
- React 19 or later (but not React 20).

The commands below use pnpm. FrameKit's runtime and CLI do not themselves require pnpm; use the equivalent commands from the package manager used by your project.

## Install the package

Install `@mauriciodmo/framekit` in your existing Next.js project:

```bash
pnpm add @mauriciodmo/framekit
```

## Configure Next.js

Open `next.config.ts` and set the `output` option to `standalone` and the `distDir` option to `.framekit/next`:

```ts
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  distDir: '.framekit/next',
  output: 'standalone',
}

export default nextConfig
```

The `standalone` output mode produces a self-contained production build. The `.framekit/next` directory is where FrameKit places generated files during development and build.

## Configure TypeScript

Open `tsconfig.json` and add the `@framekit/*` path alias inside `compilerOptions.paths`:

```json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"],
      "@framekit/*": ["./.framekit/*"]
    }
  }
}
```

This alias resolves imports from `@framekit/generated/templates` to `.framekit/generated/templates`, where the generated template registry file is written by the `framekit generate` command.

## Add styles

In your global CSS file (for example, `src/app/globals.css`), import the FrameKit stylesheet:

```css
@import "tailwindcss";
@import "@mauriciodmo/framekit/styles.css";
```

If your project does not use Tailwind CSS, omit the Tailwind import and keep only the FrameKit stylesheet import.

## Update the root layout

Open your root layout file (`src/app/layout.tsx` or `src/app/layout.jsx`). Import `FrameKitStudioRoot` from `@mauriciodmo/framekit/studio/root` and use it to wrap the `children` prop:

```tsx
import type { Metadata } from 'next'
import { FrameKitStudioRoot } from '@mauriciodmo/framekit/studio/root'

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return <FrameKitStudioRoot>{children}</FrameKitStudioRoot>
}
```

`FrameKitStudioRoot` is an async server component that emits the complete document shell: it generates or replaces `<html>`, `<head>`, and `<body>`, then renders a `FrameKitLocaleProvider` around your children. Use it directly from the App Router root layout, which must remain a server component and must not already render another `<html>`, `<head>`, or `<body>`. It reads the locale from the `accept-language` header and the user's cookie store and applies the dark or light theme class to `<html>`.

## Create the editor catch-all route

Create a catch-all route at `src/app/editor/[[...slug]]/page.tsx`. This page must be a client component:

```tsx
'use client'

import { FrameKitStudio } from '@mauriciodmo/framekit/studio'
import { templates } from '@framekit/generated/templates'

export default function EditorPage() {
  return <FrameKitStudio templates={templates} />
}
```

`FrameKitStudio` loads the template that matches the current slug, validates its definition, and renders the editor. When `slug` is empty, it shows an empty-state welcome screen.

## Redirect the root page to the editor

Open `src/app/page.tsx` (or `src/app/page.jsx`) and make it redirect to `/editor`:

```tsx
import { redirect } from 'next/navigation'

export default function HomePage() {
  redirect('/editor')
}
```

## Generate the template registry

Run `framekit generate` to discover every template in your project and write the registry file:

```bash
pnpm framekit generate
```

This command writes `.framekit/generated/templates.ts` (resolved via the `@framekit/*` alias). The generated file exports a readonly array called `templates`. Do not edit this file; it is overwritten every time you run `framekit generate` or `pnpm framekit dev`.

## The .framekit directory

The `.framekit` directory is fully disposable. It is created or regenerated by `framekit generate` and by `pnpm framekit dev`. You can safely delete it and it will be rebuilt automatically.

## Differences from a monorepo Studio setup

The FrameKit monorepo at `apps/studio` uses Turbopack's `turbopack.root` option pointing to the repository root, which requires a special Next.js configuration:

```ts
turbopack: {
  root: path.resolve(process.cwd(), '../..'),
},
```

This configuration is specific to monorepo workspaces and is not needed in a standalone project created with `create-framekit` or by integrating FrameKit manually. Your project uses the standard Next.js configuration shown above.

## Build

To create a production build, use the `framekit build` command:

```bash
pnpm framekit build
```

This command first runs `framekit check` to regenerate the template catalog and validate every template definition and locale. If validation passes, it runs `next build`. The standalone artifacts and static assets are then copied into the output directory.

## Start the production server

After a successful build, start the production server:

```bash
pnpm framekit start
```

## Environment variables

See the [CLI reference](../reference/cli.md#framekit-dev) for the authoritative distinction: `framekit dev` processes `FRAMEKIT_HOST`, `HOST`, and `PORT`, while [`framekit start`](../reference/cli.md#framekit-start) passes the inherited environment to Next's standalone server, which handles its own production variables.

## Critical limitation

All frame exports (preview, download, PNG generation) happen entirely in the browser. FrameKit does not support server-side rendering of frames, and PNG generation does not occur on the server under any configuration.
